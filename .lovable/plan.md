# CEM-10 Reports Expansion

Extend the existing Reports area into three sections — Library (default), Custom, Scheduled — built on the existing ETX2 design system (StatCard, drawers, tables, badges) and the CEM-6 compliance dataset.

## Navigation
- Add sidebar item **Delegations** (`/users/delegations`) right above Audit Trail in `AppSidebar.tsx` so the sidebar matches the spec.
- New `ReportsTabs` component (mirrors `UsersTabs`) with three tabs: Report Library, Custom Reports, Scheduled Reports.
- Routes:
  - `/reports` → Report Library (default)
  - `/reports/custom` → Custom Reports
  - `/reports/scheduled` → Scheduled Reports
  - `/reports/view/:templateId` → Report Viewer

## Data layer
New `src/data/reportTemplates.ts`:
- `ReportTemplate` (id, name, description, category, icon, lastGenerated, visualizations[], pillar, sampleKpis, sampleSeries) with the 6 templates: Utilisation, Compliance, Spend Analysis, Enquiry Activity, Approval Workflow, Delegation Activity.
- `CustomReport` list (5 seed entries: Q4 Entertainment Summary, Board Spend Overview, etc.) with owner, createdDate, lastRun, schedule, sharedWith.
- Extend `scheduledReports` in `src/data/reports.ts` with `status`, `lastRun`, `format` fields and a couple more entries.
- Helper generators for chart series derived from existing `complianceRows`, `events`, `approvalRules`, `delegations` so all charts are real mock data, not hardcoded numbers.

## Components (new)
- `src/components/reports/ReportsTabs.tsx` — tabbed nav.
- `src/components/reports/ReportLibrary.tsx` — KPI card row (6 KPIs) + template card grid (2-3 col responsive). Each card: icon, name, description, last-generated chip, Generate / Schedule buttons.
- `src/components/reports/ReportConfigDrawer.tsx` — right-side Sheet. Sections: Date Range (presets + custom), Pillar filter, User filters, Event filters, Output format. Footer: Cancel, Save Configuration, Generate.
- `src/components/reports/ReportViewer.tsx` — header (name, generatedBy, generatedOn, dataAsOf, applied filters chips, export menu), Visual/Table toggle, KPI cards, charts (recharts: bar/line/pie/donut/funnel via styled bar). Reuses existing semantic color tokens.
- `src/components/reports/SaveConfigDialog.tsx` — Name, Description, Visibility (Private/Team/Tenant), optional schedule.
- `src/components/reports/ScheduleReportDialog.tsx` — frequency, delivery date/time/timezone, recipients (chips with inactive-recipient warning + Replace/Continue), format.
- `src/components/reports/CustomReportsTable.tsx` — table with the documented columns + actions menu (Generate, Edit, Duplicate, Delete, Schedule, Export).
- `src/components/reports/ScheduledReportsTable.tsx` — columns per spec, status badge, recipients popover, run/pause actions.
- `src/components/reports/BIExportPanel.tsx` — shown inside Scheduled tab footer card: destination, last export, status, manual export, download package.
- `src/components/reports/RecentReportsWidget.tsx` — dashboard widget (Recently Generated, Scheduled, Saved, Quick Generate buttons).

## Pages
- Replace `src/pages/Reports.tsx` to render `AppShell` + `ReportsTabs` + active section based on route; keep existing `ReportsView` reachable as an "Compliance Detail" link from the Compliance template (preserves CEM-6 work).
- New `src/pages/ReportsCustom.tsx`, `src/pages/ReportsScheduled.tsx`, `src/pages/ReportView.tsx`.
- Integrate `RecentReportsWidget` into the existing dashboard (`src/pages/Index.tsx`) — add as a single new card row, no other dashboard changes.

## Charts
- Use already-installed `recharts`. Wrap in semantic tokens via `hsl(var(--primary))` etc. — no hardcoded colors.
- Visualizations per template as listed in the brief (bar, line, pie/donut, funnel-styled bar, KPI cards, heatmap as colored grid).

## Behaviour
- Generate Report: small datasets → instant viewer. Demonstrate large-dataset path with a loading state + toast "Report ready" after a short timeout for the Spend and Compliance templates.
- All exports trigger `toast.success` (mock — no real file generated).
- Permissions: respect `ActingContext` actingAs — viewer header shows "Generated by {actingAs ?? currentUser}" with `ActingOnBehalfBadge` when active.

## Out of scope
- Real PDF/CSV generation, real scheduling backend, real BI connectors, cross-tenant logic. All client-side mock.
